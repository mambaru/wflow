      80             : 
      81             :    Запросом и ответом может быть любой тип, но предпочтительно использовать структуры с определенными типами для удобства использования:
      82             : ```cpp

      84             : {
      85             :   ...
      86             : ////исключительно для удобства
      87             :   typedef std::unique_ptr<request> ptr;
      88             : };
      89             : 
      90             : struct response
      91             : {
      92             :   ...
      93             :   ////исключительно для удобства

      95             :   typedef std::function< void(ptr) > handler;
      96             : };
      97             : ```
      98             :    На вход генератора приходит ответ от предыдущего запроса, на основе которого он может создать следующий запрос. При первом запросе передается nullptr, а для того чтобы обозначить
      99             :    конец последовательности опроса генератор также возвращает nullptr. В примере простой случай, когда последовательность из одного запроса:
     100             : ```cpp
     101             : request::ptr generator(response::ptr resp)
     102             : {
     103             :   if ( resp == nullptr )

     106             : }
     107             : ```
     108             :    Далее, сгенерированный запрос передается отправителю, который должен с ним что-то сделать и отправить ответ через callback-функцию в любой момент времени (как правило асинхронно).
